Git Commands

1. git config --global user.name "Your Name"

Ustawienie globalnej konfiguracji użytkownika w Git z nazwą użytkownika.

2. git reset HEAD

Anulowanie zatwierdzenia (commit) i usunięcie zmian ze stage area.

3. gitk --all

Otwarcie interaktywnego narzędzia do przeglądania historii wizualnej.

4. git branch --merged

Wylistowanie gałęzi, które zostały scalone z aktualną gałęzią.

5. git branch "branch" "commit hash"

Utworzenie nowej gałęzi na podanym commicie.

6. git pull "remote" "branch"

Pobranie zmian z odległego repozytorium i zmergowanie ich z aktualną gałęzią.

7. git config --global user.email "your.email@example.com"

Ustawienie globalnego adresu e-mail użytkownika w Git.

8. git diff --staged

Pokazanie różnic między ostatnim zatwierdzeniem a stage area.

9. git checkout -b "branch"

Utworzenie nowej gałęzi i przejście do niej.

10. git log --graph --all

Wizualizacja historii z grafem wszystkich gałęzi.

11. git branch -d "branch"

Usunięcie lokalnej gałęzi (po upewnieniu się, że została scalona).

12. git fetch "remote" "branch"

Pobranie informacji o zmianach na odległej gałęzi, bez ich mergowania.

13. git config --global color.ui true

Włączenie kolorowania w interfejsie Git.

14. git commit "filename"

Zatwierdzenie zmian w określonym pliku.

15. git branch -v

Wylistowanie gałęzi wraz z ich ostatnimi zatwierdzeniami.

16. git push "remote" "branch"

Wysłanie lokalnych zatwierdzeń na odległe repozytorium.

17. git log "branch" --not "branch"

Wypisanie zatwierdzeń na jednej gałęzi, które nie są na drugiej.

18. git bisect (start, bad, good, reset)

Automatyczne szukanie commita, który wprowadził zmiany.

19. git commit -a -m "Commit message"

Zatwierdzenie wszystkich zmian (zarówno w stage area, jak i niezatwierdzonych) z komunikatem.

20. git add .

Dodanie wszystkich zmian do stage area.

21. git checkout "branch"

Przełączenie się na inną gałąź.

22. git remote add "remote_name" "git_url"

Dodanie odległego repozytorium pod nazwą "remote_name" z danym URL-em.

23. git blame -C "filename"

Śledzenie zmian w pliku, włącznie z przenoszeniem kodu (copy detection).

24. git branch "name"

tworzy nową gałąź o nazwie "name" w systemie kontroli wersji Git..
